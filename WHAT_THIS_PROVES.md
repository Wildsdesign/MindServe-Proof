# What This Proves

This repo exists for a technical evaluator who wants to understand what Justin
Wilds can build without receiving the private MindServe production source.

## The Short Version

MindServe proves that Justin can turn operational domain judgment into a
multi-surface software system:

- one shared operational foundation
- multiple purpose-built operator surfaces
- facility-scoped security thinking
- Azure-first deployment posture
- ADR-backed architecture decisions
- AI-assisted execution without losing human product judgment
- public proof discipline that protects private implementation value

## What To Look For

### 1. Product Judgment

MindServe is not a generic dashboard. It separates cashier, management,
ordering, and analytics jobs because those users need different interface
grammars over the same operational truth.

### 2. Architecture Judgment

The suite uses a shared foundation rather than letting every surface invent its
own business logic. That matters because inventory, ordering, cashier workflow,
reporting, and operator handoff all depend on coherent state.

### 3. Security Judgment

The public docs center facility context, scoped access, private source control,
environment-based configuration, and intentionally omitted internals. The point
is not to publish cloneable implementation details. The point is to show the
security shape of the work.

### 4. Delivery Judgment

The operating model connects domain direction, architecture, agentic execution,
repo documentation, deployment, and live validation. That loop is part of the
proof.

### 5. Boundary Judgment

This repo shows enough to evaluate the work while excluding production source,
schemas, endpoint maps, resource names, customer data, and commercially
sensitive internals.

## What This Does Not Prove

This repo does not prove every private implementation detail of MindServe. It is
not a runnable clone and should not become one.

For deeper evaluation, use this repo as a public architecture and operating
artifact, then request a guided technical walkthrough if a private review is
appropriate.
